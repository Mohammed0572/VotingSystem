# Security Policy

We take security seriously and appreciate your help in keeping this project secure. Please review our security guidelines below.

## Supported Versions

Currently, the following versions of the project are supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within this project, please **do not** open a public GitHub issue. Instead, report it securely using one of the following methods:

1. **GitHub Security Advisory (Recommended)**: Submit a draft advisory privately via the [Security tab](https://github.com/Mohammed0572/VotingSystem/security/advisories) on GitHub.
2. **Contact Maintainers**: You can reach out directly to the project maintainers through their GitHub profiles:
   * [toxicbishop](https://github.com/toxicbishop)
   * [Mohammed0572](https://github.com/Mohammed0572)
   * [supr1795](https://github.com/supr1795)
   * [Rohithgaloth](https://github.com/Rohithgaloth)

### What to Include in a Report
To help us triage and resolve the issue quickly, please include:
* A detailed description of the vulnerability.
* Step-by-step instructions (or a proof-of-concept script) to reproduce it.
* The potential impact of the vulnerability.

## Our Commitment

We will make every effort to acknowledge your report within **48 hours** and provide a resolution timeline. 

We ask that you follow **Coordinated Vulnerability Disclosure (CVD)** guidelines, keeping details of the vulnerability private until we have released a fix to protect the users of this application.
